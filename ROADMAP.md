# OpenAgentPack roadmap

This roadmap communicates direction rather than a binding delivery schedule. Priorities may
change as provider APIs evolve and the community reports real migration needs.

## Now — public beta readiness

- Keep the CLI, SDK, and Playground on one coordinated beta version.
- Make the documented quick start reproducible across every supported provider.
- Publish provider capability differences as native, emulated, or unsupported.
- Keep CI, package compatibility, security scanning, and release provenance green.

## Next — portability proof

- Publish end-to-end migration examples between managed-agent providers.
- Expand provider conformance tests and document live-test coverage.
- Improve import, drift explanation, and destructive-change safeguards.
- Create contributor-ready issues for providers, documentation, and examples.
- Publish a searchable documentation site without changing the repository-first workflow.

## Later — ecosystem

- Stabilize the provider extension interface.
- Build a reusable catalog of agent declarations, skills, and deployment patterns.
- Add remote and team-oriented state backends while preserving local-first operation.
- Define the compatibility and deprecation policy required for a stable `1.0` release.

## Shape the roadmap

- Ask questions or propose designs in [GitHub Discussions](https://github.com/modelstudioai/OpenAgentPack/discussions).
- Request a provider or capability through the [feature request form](https://github.com/modelstudioai/OpenAgentPack/issues/new?template=feature-request.yml).
- Follow accepted work in [GitHub Issues](https://github.com/modelstudioai/OpenAgentPack/issues).
