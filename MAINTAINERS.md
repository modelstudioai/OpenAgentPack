# Maintainers and Governance

OpenAgentPack is maintained by the `@modelstudioai/openagentpack-maintainers` GitHub team.

## Decision making

- Routine changes are accepted through pull requests after CI passes and at least one maintainer approves.
- Changes to public APIs, configuration compatibility, provider contracts, security controls, or the release process require an explicit design rationale in the pull request.
- Maintainers resolve disagreements by seeking consensus first. If consensus cannot be reached, the maintainers responsible for the affected subsystem make the final decision and document it in the pull request.

## Releases and security

Only maintainers may publish packages or create releases. Security reports follow the private process in [SECURITY.md](./SECURITY.md).

Maintainer access is reviewed periodically and removed when it is no longer needed.
